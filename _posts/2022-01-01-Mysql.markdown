### 排序：

#### 规则：

ASC：升序，DESC：降序，ORDER BY 放在最后。

```mysql
SELECT employee_id from BIAO ORDER BY employee_id DESC;
#默认升序，未指明的情况。
```

#### 二级排序：

```mysql
SELECT employee_id，salary 
from BIAO 
ORDER BY employee_id DESC，salary ASC;
```

##### 注意：

别名只能在ORDER BY中使用，不能再WHERE中使用。并且WHERE必须写在FROM后，ORDER BY之前。

### 分页：

##### 解决问题：

查询返回的记录太多查看不方便，表里多条数据指向显示第x条

#### 规则;

```mysql
#每页显示20条,此时第一页
SELECT employee_id，salary 
from BIAO 
LIMIT 0,20;
#每页显示20条,此时第二页
SELECT employee_id，salary 
from BIAO 
LIMIT 20,20;

```

### 多表查询：

##### 解决问题：

每个查询语句不会针对一个表，所以需要多表查询。

#### 规则：

```mysql
#这样得到的并不是想要的多表查询，反而是笛卡尔积之后的数据。（交叉链接）{有点文本匹配数据集的意思}
SELECT a,b
FROM biao1,biao2

#正确多表查询：
SELECT a,b
FROM biao1,biao2
WHERE biao1.元素 = biao2.元素
```

#### 分类：

1. 等值链接 VS 非等值链接
2. 自链接 VS 非自链接
3. 内连接 VS 外连接（查所有就是外连接）

ON E.`DEPARTMENT_ID` = D.`DEPARTMENT_ID`
UNION ALL
SELECT EMPLOYEE_ID,DEPARTMENT_NAME
FROM EMPLOYEE E LEFT JOIN DEPARTMENTS D
ON E.`DEPARTMENT_ID` = D.`DEPARTMENT_ID`
WHERE E.`DEPARTMENT_ID` IS NULL

#### 聚合函数：

输出一组数据，输出单个值。

##### 注意：

聚合函数不能嵌套使用。

##### 常见聚合函数：

```mysql
#AVG/SUM 适用于数值类型的求总和
SELECT AVG(salary)
FROM employees；

SELECT SUM(salary)
FROM employees；

#MAX/MIN 适用于字符串，数值，时间日期类型
SELECT MAX(salary),MIN(salary)
FROM employees；

#COUNT
#作用：计算指定字段在查询结果中出现的个数(算人头的，有一个算一个，和数值没关系)
SELECT COUNT(salary)
FROM employees；
#COUNT(1)是把每一行都看作1，所以可以用COUNT(1)、COUNT(*)来查看有多少条记录。
#划重点：COUNT计算时不计算空值。





#方差标准差中位数

```

##### GROUP BY：

```mysql
#查询不同部门平均工资
SELECT AVG(salary)
FROM employees
GROUP BY department_id;
#SELECT 声明的必须出现在GROUP BY 中，反之不必须
#WITH ROLLUP 计算整体
```

##### HAVING关键字：

```mysql
#过滤数据,如果过滤条件中使用了聚合函数，那么必须使用HAVING替代WHERE 
SELECT department_id,MAX(salary)
FROM employees
GROUP BY department_id
HAVING MAX(salary)>10000;
#过滤条件中有聚合函数时使用having，没有时也可以，但是建议用where，效率高
```
